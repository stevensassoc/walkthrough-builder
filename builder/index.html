Getting started

  1. Drag 360° panorama JPGs onto ＋ Drop panoramas (left). They're auto-resized to the chosen Quality.
  2. Click an area to preview it; drag the photo to look around, scroll to zoom.
  3. Rename areas inline; drag them to reorder (this sets the Next/Back order clients follow).
  4. + Link, then click the photo to point to another area — set its target and aim its arrow with the direction slider. + Note adds an info note.
  5. Drag any placed hotspot to reposition it; click one to edit or delete it.
  6. Aim the view and press Set start view so each area opens facing the right way.
  7. Save keeps the project in this browser and downloads a project file. Export ZIP produces the finished tour folder — it must be hosted on a web server (not double-clicked); see the note shown after exporting.

If you just double-click index.html, the buttons load but the 360° photos stay blank, because browsers block local files from loading into the 3D view for security.

To view or share it

  1. Publish it — upload the unzipped folder to any static web host (e.g. GitHub Pages). It works at that URL and you can share the link. The upcoming Publish feature will do this for you.
  2. Preview locally — in a terminal in the folder, run python -m http.server 8000 and open the address it prints. (See the HOW TO PUBLISH THIS TOUR.txt inside the ZIP.)

Publish settings

Publishing uploads tours to your firm's GitHub Pages site. You need a GitHub personal access token (classic) with the repo scope. Create one ↗

The token grants write access to the org's repos and is stored only in this browser. Use it on trusted machines and revoke it on GitHub if a machine is lost. It is sent only to api.github.com.
